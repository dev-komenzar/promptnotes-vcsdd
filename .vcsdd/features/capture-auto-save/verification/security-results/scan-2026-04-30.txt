== Manual Review Findings ==

1. Input handling:
   - `serializeNote` takes a ValidatedSaveRequest (already validated by Step 1).
   - Tag values are inserted into YAML via string interpolation.
   - No HTML escaping needed — output is Markdown/YAML, not HTML.
   - Malicious tag values (e.g., containing "---") cannot inject additional YAML
     delimiters because the YAML serializer wraps them as list items ("  - <tag>").
     Confirmed by fast-check property test (PROP-002) finding counterexample
     "a---" and verifying the regex /^---\n[\s\S]*\n---\n[\s\S]*$/ still holds.

2. File path construction:
   - Path constructed as `${infra.vaultPath}/${validatedRequest.noteId}.md`
   - NoteId is a branded type — callers are responsible for ensuring it is safe.
   - No path traversal sanitization is performed in the domain layer; this is
     correctly deferred to the Vault Context (Rust) per architectural boundary.

3. Atomic writes:
   - `FileSystem.writeFileAtomic` is a port — implementation performs temp → rename
     for crash safety. The domain layer does not implement the atomic write itself.

4. Event publication:
   - Events are plain data objects; no deserialization from untrusted input.
   - `publish()` is a port — domain layer emits typed events only.
