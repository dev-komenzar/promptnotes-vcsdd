=== PATH TRAVERSAL SCAN ===
Target: pipeline.ts line 124
  const filePath = `${infra.vaultPath}/${validatedRequest.noteId}.md`;

NoteId format (docs/domain/code/rust/src/value_objects.rs L254-L287):
  Pattern: YYYY-MM-DD-HHmmss-SSS[-N]
  Allowed chars: ASCII digits [0-9] and hyphens [-]
  Smart Constructor: rejects all other characters including '/', '.', '\', ' '
  Result: path traversal via noteId is STRUCTURALLY IMPOSSIBLE

VaultPath validation (docs/domain/code/ts/src/shared/value-objects.ts L151-L156):
  Smart Constructor: rejects empty strings and non-absolute paths
  Result: VaultPath cannot be relative — reduces (but does not eliminate) path manipulation risk

FINDING: No runtime validation of noteId in TypeScript layer — relies entirely on Rust Smart Constructor.
The TypeScript domain treats NoteId as an opaque branded type received from Rust.
Risk: ACCEPTED (Rust layer enforces invariant; domain boundary clearly documented)

Risk: LOW (not currently exploitable via normal UI flow; Tag Smart Constructor is a Phase 11+ item)

=== CONTENT ESCAPE / PARAGRAPH --- COLLISION ===
Target: serialize-blocks-to-markdown.ts serializeBlock()
  paragraph case: returns raw content (no prefix, no escaping)

If a paragraph block has content exactly "---":
  serializeBlocksToMarkdown([{type:"paragraph", content:"---"}]) === "---"
  parseMarkdownToBlocks("---") produces [{type:"divider"}]
  Roundtrip FAILS: paragraph -> divider (structural type change)

PROP-026 arbParagraphBlock filter:
  const MARKDOWN_PREFIXES = /^(# |## |### |- |1\. |> |---$|```)/;
  Paragraph content "---" is excluded from property test inputs
  This documents the constraint but does NOT fix it

FINDING: Paragraph block content starting with "---" (exact) or any Markdown prefix
produces a lossless-parse violation. This is a KNOWN LIMITATION documented in the
PROP-026 harness. The property test generator correctly avoids this case.
Risk: MEDIUM for data integrity (affects users who type exactly "---" as paragraph text).
     No security exploitability (affects only the user's own note content).
     Tracked as advisory finding FIND-021 (open, from Phase 3 adversarial review).
