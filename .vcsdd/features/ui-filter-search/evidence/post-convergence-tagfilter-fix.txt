Defects addressed (rooted in ui-tag-chip)
-----------------------------------------
1. TagFilterSidebar.svelte rendered `entries` in caller-supplied order, but
   REQ-TAG-009 / PROP-TAG-012 specify "entries sorted by usageCount descending"
   as a property of THE SYSTEM (the sidebar component itself). Tests passed
   un-sorted input expecting the component to enforce ordering.
2. The clear-all button's data-testid was `tag-filter-clear` while the DOM
   test fixture queried `tag-filter-clear-all`. The rendered button never
   matched, causing 4 of the 5 failures.

Coherence resolution
--------------------
- Component (promptnotes/src/lib/feed/TagFilterSidebar.svelte):
    * Added `sortedEntries = $derived([...entries].sort((a,b) => b.usageCount - a.usageCount))`
    * Iterated `sortedEntries` in the {#each} block instead of `entries`
    * Renamed clear button data-testid: `tag-filter-clear` → `tag-filter-clear-all`
- Findings annotated with resolution metadata:
