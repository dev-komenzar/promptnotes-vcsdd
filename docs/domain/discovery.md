# Discovery

## ビジネス概要

AI を使う人（特に Claude Code などのプロンプト系ツール利用者）向けの「**下書きジャーナル**」。

書く瞬間は摩擦ゼロで本文に集中でき、振り返る時は frontmatter（タグ等）で整理できる。Markdown ファイルとして保存し、Obsidian など外部ツールと共存可能。**「書いたものが消えない安心感」** をユーザーに提供する。

想定する痛み:
- AI ツールで誤って Enter を押して送信してしまう事故
- 「前にうまくいったプロンプト」が思い出せない／消えてしまう不安
- Obsidian は新規メモ作成時の摩擦が大きい（タイトル・frontmatter 入力など）

## Core Domain

**名前**: Frictionless Capture × Stockable Structure（書く摩擦ゼロ × 後から整理可能な構造）の両立

**理由**: 「Capture（流す）」と「Curate（貯める・整理する）」の連続的な体験を 1 アプリで提供することが、Obsidian（Stock 寄り）や軽量メモ帳（Capture のみ）に対する差別化になる。

**差別化要因**:
- 起動・新規作成の摩擦ゼロ
- frontmatter は背景処理しつつ、振り返り時には前面化
- Obsidian と vault を共有可能（依存ではなく共存）

### Stream Mode（Capture：書く）

**中心価値**: 摩擦ゼロで書ける／本文に集中できる／誤送信からの避難所

**振る舞い**:
- タイトルはタイムスタンプから自動決定（ユーザーは命名しない）
- frontmatter は背景で生成（ユーザーは触らない）
- アプリを開いた瞬間、**カーソルが入力可能な位置にある**
- 本文だけを書ける UI
- ワンクリックで本文をコピー
- 自動保存（一定時間 or フォーカスアウトでファイル化）
- Logseq ジャーナル風の時系列フィード表示

### Stock Mode（Curate：振り返る・整理する）

**中心価値**: 過去の下書きを振り返り・分類・再利用できる足場

**振る舞い**:
- frontmatter（タグなど）でフィルタ・ソート
- ファイル名（タイムスタンプ）での時系列ソート
- 振り返りはこのアプリ内で完結する（外部ツール起動を強制しない）

## Supporting Subdomains

- **Markdown Persistence**: Markdown ファイルとして書き出す。Obsidian 互換性を担保するため frontmatter は YAML で先頭に配置。
- **Frontmatter Management**: YAML フロントマター生成・編集。MVP では固定スキーマ。将来的にユーザー定義テンプレートを許す方向。
- **Clipboard Integration**: 本文ワンクリックコピー。frontmatter を含めずに本文のみコピーする責務。
- **Feed Rendering**: 時系列ジャーナル UI（Logseq Journal 風）。
- **Auto Save**: 一定時間経過 or フォーカスアウト等のトリガで永続化。

## Generic Subdomains

- **Markdown Editor**: 既存 OSS ライブラリで代替可能。
- **File System I/O**: OS のファイルアクセス層。
- **Timestamp Formatting**: 日時整形（ファイル名生成用）。

## スコープ外（明示的に除外）

- **下書きの「使った／未使用」状態管理**: ユーザーが外部に投げたかどうかは追跡しない。
- **MVP でのユーザー定義 frontmatter テンプレート**: 望ましいが MVP には含めない（固定スキーマで開始）。

## 未解決の問い（次フェーズに持ち越し）

- フィードはどの粒度で時系列を区切るか（日単位ジャーナル？フラットなストリーム？）
- 編集途中のメモと「保存済み」メモの境界はあるのか
- frontmatter の固定スキーマには何を含めるか（タグ・作成日時・更新日時・モデル名 など）
- vault フォルダ（保存先ディレクトリ）の決定方法（設定 / 既定値 / ワークスペース概念の有無）
- 自動保存のトリガ詳細（何秒、フォーカスアウトの定義）
- ストリームと個別ノートの編集 UI は同一画面かページ遷移か
